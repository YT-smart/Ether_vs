# 维护者打包发布说明

这个文档给项目维护者或打包发布的人看，普通用户只需要阅读 `README.md`。

## 用户使用入口

1. 解压整个工具目录。
2. 双击 `启动.bat`。
3. 工具会自动检查环境；缺少依赖或模型时会先自动安装。
4. 安装完成后，浏览器会自动打开 `http://127.0.0.1:3666/`。

## 用户电脑需要提前安装

- Windows 10/11
- Python 3.11
- Node.js 20+
- ffmpeg

启动脚本会自动检测 `.venv`、Python 依赖和 Qwen3-ASR 0.6B 模型是否完整；不完整时会调用安装脚本。安装脚本会自动创建 `.venv`，并默认使用国内镜像安装 Python 依赖、CUDA 版 torch 和 Qwen3-ASR 0.6B 模型。

## 默认下载源

- Python 包：清华 PyPI 镜像
- CUDA 版 torch：阿里云 PyTorch wheels
- Qwen3-ASR 模型：优先 ModelScope，失败后尝试 HuggingFace 国内镜像

## 打包时建议包含

```text
server.mjs
package.json
public/
scripts/
tools/
安装依赖.bat
启动.bat
install.bat
打包发布说明.md
```

## 打包时可以不包含

```text
.venv/
models/
downloads/
work/client-jobs/
work/wheels/
skills/
douyin-video-breakdown/
outputs/
.agents/
.codex/
.clawhub/
.git/
server.js
__pycache__/
```

这些内容用户可以通过 `安装依赖.bat` 自动生成或下载。

`skills/` 和 `douyin-video-breakdown/` 是开发过程里用到的 Codex skill/草稿目录，运行工具不需要。运行时只需要 `tools/` 里的解析脚本、`parse-video-py` 源码和 ASR 脚本。

## 可选：加速 torch 安装

如果你想让用户安装更稳，可以把这个文件也放进 `work/wheels/`：

```text
torch-2.11.0+cu126-cp311-cp311-win_amd64.whl
```

安装脚本会优先使用本地 wheel；没有本地 wheel 时才从阿里云下载。

## GPU / CPU

工具会自动识别：

- 有可用 NVIDIA CUDA 环境：使用 GPU
- 没有 CUDA：使用 CPU

CPU 也能运行，但提取文案会明显更慢。
